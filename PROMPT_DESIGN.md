# 프롬프트 설계 문서

`src/message_generator.py`의 시스템 프롬프트 및 데이터 포맷 설계 결정 기록입니다.

## 설계 목표

- 카카오톡 메시지에 최적화된 **짧고 읽기 쉬운** 브리핑
- 하루 일정(집 → 출근 → 회사 → 퇴근 → 집)을 반영한 **실용적 정보**
- Claude가 판단할 수 있도록 **구조화된 데이터** 전달

---

## 시스템 프롬프트 설계

### 스케줄 컨텍스트

```
- 첫 번째 위치(집): 아침 출근 전 및 저녁 퇴근 후 기준
- 두 번째 위치(회사): 오전 9시 ~ 오후 6시 기준
```

**이유**: 위치가 여러 개일 때 Claude가 어느 데이터를 언제 기준으로 판단해야 하는지 알 수 없어서 명시적으로 지정. 없으면 첫 번째 위치만 기준으로 삼거나 임의로 판단하는 경향이 있었음.

### 우산 권고 기준

```
출근(7~9시) 또는 퇴근(18~20시) 강수확률이 50% 이상이면 "우산 챙기세요"
```

**이유**: 하루 최대 강수확률 기준으로 하면 점심시간 잠깐 오는 비에도 우산을 권고하게 됨. 실제로 우산을 챙겨야 하는 시간대(출퇴근)만 기준으로 삼도록 제한.

### 옷차림 기준

```
집의 아침 최저기온 + 회사의 낮 최고기온을 모두 고려해 결정
```

**이유**: 하루 최저/최고 기온만 주면 Claude가 한 시점 기준으로만 판단하는 경우가 있음. 아침 출근 시점(집 기준 최저)과 낮 시간(회사 기준 최고)을 모두 고려해야 레이어드 추천 등이 자연스럽게 나옴.

### 옷차림 8단계

| 기온       | 추천                                    |
| ---------- | --------------------------------------- |
| 28°C 이상  | 민소매·반소매·반바지·치마·린넨          |
| 23~27°C    | 반소매·얇은 셔츠·반바지·면바지          |
| 20~22°C    | 블라우스·긴소매·면바지·슬랙스           |
| 17~19°C    | 얇은 가디건·니트·스웨트셔츠·후드·긴바지 |
| 12~16°C    | 자켓·가디건·니트·스타킹·청바지          |
| 9~11°C     | 트렌치코트·야상 점퍼·스타킹·기모바지    |
| 5~8°C      | 울코트·히트텍·가죽 옷·기모              |
| 4°C 이하   | 패딩·두꺼운 코트·누빔 옷·기모·목도리    |

**이유**: Claude에 기온만 주면 추상적인 추천("따뜻하게 입으세요")을 하는 경향. 구체적인 의류 목록을 제공하면 실용적인 추천이 나옴. 선택지를 많이 줄수록 기온·성별·스타일을 고려한 다양한 조합이 가능.

### 글자 수 제한: 400자

**이유**: 300자로 시작했으나 위치가 두 곳일 때 날씨 요약 + 우산 + 마스크 + 옷차림 + 응원을 모두 담기에 부족. 400자로 늘렸을 때 내용이 자연스럽게 맞아떨어짐.

## 데이터 포맷 설계 (`_format_data`)

Claude에 전달하는 데이터는 단순 수치가 아니라 **판단에 필요한 맥락**을 함께 전달합니다.

### 위치별 역할 명시

```
[집 / 집 — 아침·저녁 기준]
[회사 / 회사 — 9~18시 기준]
```

위치 이름만 주면 역할을 모르므로 데이터 레이블에 역할을 함께 표기.

### 출퇴근 강수확률 분리

```
- 출근 강수확률 (7~9시): 30%
- 퇴근 강수확률 (18~20시): 60%
```

하루 최대 강수확률 하나만 주면 Claude가 출퇴근 시간대를 구분할 수 없음. 시간대별로 분리해서 "퇴근길 우산" 같은 구체적 안내가 가능해짐.

### 아침 최저기온 / 낮 최고기온 분리

```
[집]
- 아침 최저기온: 11°C

[회사]
- 낮 최고기온 (9~18시): 21°C
```

일 최고/최저만 주면 언제 그 기온이 나오는지 Claude가 모름. 집은 아침 기준, 회사는 낮 기준으로 분리해서 레이어드 추천의 정확도를 높임.

## 향후 개선 방향

- **사용자 피드백 반영**: "오늘 브리핑 너무 길었어" 같은 피드백을 프롬프트에 반영하는 동적 조정
- **계절별 프롬프트 분기**: 여름/겨울 특화 안내 (자외선, 동파 등)
- **Claude가 데이터 이상 감지**: 누락된 시간대, 비정상 수치 등을 Claude가 판단해 처리하는 Agent 구조로 발전
